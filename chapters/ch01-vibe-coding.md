# Chapter 1｜什麼是 Vibe Coding？

## Vibe Coding 前言

Vibe Coding（中文常譯為「氣氛編程」或「氛圍編碼」）是一種新型的軟體開發工具，核心概念是開發者不再親手撰寫每一行代碼，而是透過自然語言與 AI 溝通，讓 AI 負責實作邏輯與修正 Bug。「Vibe coding」（氣氛編碼）是一種由 Andrej Karpathy 於 2025 年提出的 AI 輔助開發範式「寫程式的新方式」，意思是不再一行一行的輸入程式碼，而是直接用「講的」，請 AI 把程式碼寫出來。開發者以自然語言描述需求，讓大型語言模型（LLM）自動生成程式碼，重點在於透過聊天介面「用文字說出我的需求」來與AI互動，只要你講的夠清楚，AI 就能幫你生出畫面、程式碼、流程與技術文件。透過測試回饋與調整，而非專注於編寫複雜語法，顯著降低了開發門檻。

## Vibe Coding、LLM Coding、傳統編碼哪裡不一樣？
| 項    目 | 傳統編碼 | LLM Coding | Vibe Coding |
|----|----------|------------|-------------|
| **操作方式** | 手動撰寫程式碼 | 人為主，AI 協助 | 由 AI 產生出程式碼 |
| **寫程式碼** | 大量的手寫程式 | 中等、提供建議與部分程式碼 | 可以完全不需要或低度參與 |
| **AI 參與度** | 沒有或很少 | 中等、還是需要自己除錯 | 極高、可讓 AI 全部完成 |
| **學習門檻** | 極高，要懂語法懂邏輯 | 中等、需有基本知識 | 極低、專注在提示詞的學習 |
| **開發速度** | 慢 | 中等，AI 分攤掉一些工作量 | 極快、就幾分鐘的事 |
| **錯誤處理方式** | 系統回饋錯誤，自己手動改 | AI 的角色是協助與解釋 | 測試結果給 AI，AI 幫你再修正 |
| **維護與擴充性** | 高 | 需人工整合與調節 | 低、程式結構難穩定 |
| **程式碼品質** | 架構較完整與穩定 | 取決提示語與理解的深度 | 反覆修正 |

## Vibe Coding 的核心觀點與特點
- 定義： 使用自然語言描述功能（如「幫我寫個剪刀、石頭、布的遊戲」），AI 已經在你之前研讀與熟悉剪刀、石頭、布的規則，
         你只要說出「幫我寫個剪刀、石頭、布的遊戲」，你只要等待 AI 給的結果再逐一的。
- 核心精神： 順應感覺 (Vibes)、專注於在你想到得到什麼，不需要在意相關技術的細節。只要能清楚的表達你的想法，AI 就能幫你把產品建構出來。
- 優勢： 零程式基礎也能入門、快速原型製作（MVP）、開發初期的成本顯著下降。
- 局限： 產出的程式碼品質不一、程式碼品質難以掌控、除錯困難仍需人工審核。效能與維護困難且仍存在安全風險，需進行人工最後把關。 




零程式基礎也能入門：使用者僅需描述需求，AI 自動撰寫邏輯與功能。
快速原型開發：對初創團隊與個人創作者特別有利，可節省大量時間成本。
創意落地更容易：設計師與PM也能參與技術開發流程，加速驗證與測試。
開發成本顯著下降：適用於預算有限的項目、個人專案與教育領域。


寫程式很少是線性思考，而比較像這樣：

1. 想到一個模糊的念頭
2. 寫下第一行不太確定的程式
3. 直接執行
4. 看回饋
5. 再修一點點

這個循環，就是所謂的 vibe。

## 什麼是 Vibe Coding？

Vibe Coding 不是一種語言，
而是一種「你和電腦之間的互動節奏」。

> 💡 重點
>
> 當節奏順，程式自然會長出來。
